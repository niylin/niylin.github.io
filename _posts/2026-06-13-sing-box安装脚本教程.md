# 使用mnc脚本快速搭建教程
[仓库地址]()
## 服务端使用 sing-box

### 安装依赖: curl,bash,大多数系统镜像已包含跳过即可
```bash
if command -v apt >/dev/null 2>&1; then
    apt update -y
    apt install -y curl bash
elif command -v dnf >/dev/null 2>&1; then
    dnf install -y curl bash
elif command -v yum >/dev/null 2>&1; then
    yum install -y curl bash || true
elif command -v pacman >/dev/null 2>&1; then
    pacman -Sy --noconfirm curl bash
elif command -v apk >/dev/null 2>&1; then
    apk add --no-cache curl bash
else
    echo "错误：无法识别包管理器，请手动安装 curl。"
    exit 1
fi
```
### 运行脚本

```bash
curl -fsSL -o sing-box-install.sh https://raw.githubusercontent.com/niylin/mnc-install/master/sing-box-install.sh && chmod +x sing-box-install.sh && ./sing-box-install.sh
```

- 正常输出如下
```

检测到位置: 🇺🇸 US (IAD)
IP 是否正确？默认 y，输入 n 可手动覆盖:

  AnyTLS TCP + TUIC UDP: 51112
```
1. 如果VPS使用IPv6入站,则输入6,使用IPv4确认即跳过
2. 由于部分NET VPS 网卡上并没有真实IP,会通过请求公网 `https://cloudflare.com/cdn-cgi/trace` 来获取IP,请求完成会返回IP信息,如果输出的IP正确跳过即可,如果不对,输入n,然后输入IP进行覆盖.
大多数情况下可以获取正确IP,在VPS上出口和入口不一致的情况下需要手动输入,因为获取的是出口IP,比如宿主机配置了warp出站   
3. 输入一个端口,默认情况下需要两个连续端口,输入内容和其+1,包含TCP和UDP,NET VPS建议先配置玩端口转发再运行脚本

- 交互部分基本只有这些,运行结束会输出节点信息,根据设备性能和带宽,整个安装流程大约在 3-30s之间

proxy-providers:

    url: https://jimmy-others-invited-catalogue.trycloudflare.com/598138fa-d96d-4b69-a11f-43288d1c5da1/20260613-203933.yaml

2. 客户端 sing-box 配置:可直接导入 sing-box 相关客户端使用,比如sing-box for Android 和nekobox 等支持原始sing-box配置的客户端,如果 v2rayn支持导入 sing-box json格式的配置,应该也可以使用

### 中转用法
1. 在落地VPS上运行
```
curl -fsSL https://raw.githubusercontent.com/niylin/mnc-install/master/sing-box-install.sh | bash -s -- -ouserver
```

用法: sing-box-install.sh [参数]

  安装 cloudflared、sing-box、nginx，配置四个直连协议和一个 VMess-ws 临时隧道节点。
